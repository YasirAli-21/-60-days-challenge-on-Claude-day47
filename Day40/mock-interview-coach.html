System prompt design

The assistant is instructed to act as a friendly-but-honest interview coach for entry-level candidates. It runs a strict state machine with four output types — clarify, question, feedback, summary — returned as JSON only, so the UI never has to parse free-form prose. Each answer is scored 1–10 with concrete strengths, concrete improvements, and a rewritten "tightened" example sentence, which turns vague praise into something the candidate can actually reuse. A STAR-method nudge is included only for behavioral questions, since it isn't relevant to motivational or role-fit questions.

Edge cases are handled explicitly in the prompt: nonsense or empty job titles trigger a clarify response instead of guessing; abusive or off-topic input gets a short, boundary-setting decline and steers back to practice; a blank or one-word answer is scored honestly and flagged as too short rather than inflated.

Why the UI choices fit this use case

  • Backstage / "green room" concept instead of a generic chat box — interview prep is inherently about calming pre-performance nerves, so the visual language borrows from a theatre green room: warm gold spotlight accent, dark ink backdrop, calm sage for "what's working."
  • Segmented progress track instead of a chat scrollback, so a nervous first-time candidate always knows exactly how many questions remain — reducing anxiety was the stated goal, not just data collection.
  • Score dial + strengths/improvements split mirrors how a real recruiter debrief is structured, and separates "keep doing this" from "fix this" so the encouraging tone doesn't blur into empty praise.
  • One question at a time matches the requested interaction model exactly — no wall of text, no multi-question forms.

Extending this

  • Add memory: persist each session's transcript + summary (e.g. to localStorage or a backend) so returning users can track score trends across sessions instead of starting cold each time.
  • Add tools: wire in a real job-postings API or web search so the assistant grounds questions in an actual live job description rather than an inferred role.
  • Multi-step flows: branch difficulty — if a candidate scores consistently high, escalate to harder follow-up/curveball questions; if low, offer a "see a strong sample answer" step before retrying.
  • Voice mode: swap the textarea for speech-to-text to simulate the verbal pressure of a real interview, then feed the transcript through the same feedback JSON contract.
